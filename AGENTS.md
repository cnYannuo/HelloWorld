# AGENTS 协作说明

## 项目概览
- 项目名称：`HelloWorld`
- 当前状态：极简示例仓库（参考 `README.md`）

## 主要目标
- 保持代码与文档简洁、可读、可维护。
- 在不引入不必要复杂度的前提下完成用户需求。
- 优先做最小可行改动（small, focused changes）。

## 代理工作原则
1. **先理解再修改**：先阅读相关文件，避免盲改。
2. **最小范围改动**：只修改完成任务所需的文件和代码段。
3. **保持风格一致**：遵循现有命名、结构与格式。
4. **避免破坏性操作**：不随意删除文件、不重置历史、不覆盖用户已有改动。
5. **可追溯**：提交信息清晰描述“做了什么、为什么做”。

## 执行流程
1. 明确需求与验收标准。
2. 定位并阅读相关文件。
3. 实施改动（优先单一职责、低耦合）。
4. 进行必要验证（如 lint/test/运行检查）。
5. 更新文档（如行为变化、使用方式变化）。
6. 提交并推送改动。

## 代码与文档规范
- 默认使用 ASCII；仅在确有必要时使用非 ASCII 字符。
- 注释应解释“为什么”，避免解释显而易见的“做什么”。
- 若新增依赖，使用包管理器安装最新稳定版本。

## Git 规范
- 每个逻辑变更使用独立提交。
- 提交信息建议格式：
  - `feat: ...` 新功能
  - `fix: ...` 缺陷修复
  - `docs: ...` 文档更新
  - `refactor: ...` 重构
  - `chore: ...` 杂项维护

## 验证要求
- 若仓库存在测试或检查命令，至少执行与本次改动相关的最小验证。
- 无自动化测试时，提供简要人工验证说明。

## 沟通要求
- 对外沟通应简洁明确，说明：
  1. 改了什么；
  2. 为什么这样改；
  3. 如何验证；
  4. 是否存在后续建议。
